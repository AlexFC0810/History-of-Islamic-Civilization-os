# Load-Bearing Facts

## Purpose

This file defines the difference between ordinary historical detail and load-bearing historical intelligence.

The steward should not treat all facts as equal.

## Definition

A **load-bearing fact** is a fact that changes the map.

It helps explain:

- why a civilization rose;
- why a system worked;
- why an institution endured;
- why a narrative became distorted;
- why a decline occurred;
- why a renewal strategy might work.

## Non-load-bearing trivia

A fact may be interesting but not yet load-bearing if it does not explain a larger pattern, correct a false map, or unlock reimplementation.

Trivia can become useful later, but it should not dominate synthesis.

## Load-bearing test

Before emphasizing a fact, ask:

1. Does this explain multiple downstream outcomes?
2. Does this reveal a causal loop?
3. Does this correct a common false map?
4. Does this expose a missing variable?
5. Does this show an institutional mechanism?
6. Does this change the timeline?
7. Does this clarify incentives?
8. Does this distinguish myth from reality?
9. Does this create a practical reimplementation lesson?
10. Would the map be weaker without it?

## Examples of load-bearing categories

- A translation movement is load-bearing because it explains knowledge transfer.
- Waqf is load-bearing because it explains durable public funding.
- Anti-riba ethics are load-bearing because they reveal a moral-financial operating principle.
- Isnad is load-bearing because it reveals information integrity architecture.
- Bimaristans are load-bearing because they show mercy, medicine, training, and public welfare becoming institutions.
- Colonial mandates are load-bearing because they explain borders, dependency, and state formation constraints.

## Default rule

> **Do not ask only, “Is this true?” Ask, “What does this truth carry?”**

## Final line

> **The steward’s job is not to know every fact. It is to find the facts that hold up the map.**
